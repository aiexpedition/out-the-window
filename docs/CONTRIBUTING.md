# CONTRIBUTING

This project uses multiple AI collaborators with clearly defined roles.  
Contributions must respect these boundaries to preserve system integrity, velocity, and canon accuracy.

`/docs` is the source of truth.  
The codebase reflects decisions; it does not define them.

---

## Team Model & AI Collaboration

Work is divided by **authority**, **affordance**, and **intent** — not by convenience or technology stack.

### Roles

#### Dan (Product Owner & Intent Authority)
Owns product intent and acceptance criteria.

Responsibilities:
- Define business goals and product direction for Out the Window
- Set refusal boundaries and tradeoff tolerance
- Decide when work is complete and ready to ship

Answers:  
**"What are we trying to accomplish, and when is it good enough?"**

---

#### Claude (Chief Architect & Canon Editor)
Owns system architecture, workflow sequencing, and canon integrity.

Responsibilities:
- Decompose objectives into executable work
- Orchestrate execution via CC (Claude Code) or direct code generation
- Produce **downloadable, role-specific prompts** for CC when needed
- Generate code directly for small changes (≤2 files)
- Ensure `/docs` accurately reflects how the system actually works
- Maintain consistency between implementation, canon, and context capsules

All structural, architectural, or process changes must be reflected in `/docs`.

Answers:  
**"How should this system be built and evolved safely?"**

---

#### CC — Claude Code (Execution Agent)
Autonomous code execution with file system access.

Responsibilities:
- Execute prompts generated by Claude (Chief Architect)
- Make systematic changes across multiple files
- Implement backend and frontend code
- Provide iterative refinement with error recovery
- Report execution summary back to Claude for review

Answers:  
**"I'll execute this plan and report back what changed."**

---

Historical context capsules and archived prompts may reference legacy agent names. Treat any such references as mapping to the current role equivalent using the Agent Reassignment Map below.

## Contribution Rules

- Do not cross role boundaries without explicit instruction
- Authority decisions precede affordance decisions
- All delegated work must originate from **Claude** as a downloadable markdown prompt
- **Every prompt must include a recommended git commit message**
    - Imperative mood
    - Scoped to the change
- Do not introduce new conventions without documenting them in `/docs`
- Prefer explicit file replacements or diffs over vague descriptions
- Assume `/docs` is the source of truth, not the codebase alone

If something is unclear, stop and ask rather than inventing behavior.

---

## Guiding Principle

Conversation is where ideas form.  
Canon is where truth stabilizes.

If it matters tomorrow, it belongs in `/docs`.
